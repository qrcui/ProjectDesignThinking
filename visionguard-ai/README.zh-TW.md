# VisionGuard AI

[English](README.md) | [简体中文](README.zh-CN.md) | **繁體中文** | [儲存庫首頁](../README.md)

VisionGuard AI 是一款基於瀏覽器的視力篩檢與數位視疲勞監測 MVP。它使用裝置攝影機在本地執行臉部和眼部關鍵點推理，估算相對觀看距離、統計眨眼、提供啟發式疲勞指標，並引導使用者完成 Landolt C 缺口方向測試。

> **重要提示：** 本專案僅用於健康篩檢和產品展示，不能診斷近視、散光、眼病或其他醫療狀況，也不能代替驗光師或眼科醫師的檢查。如持續出現視物模糊、眼痛、復視、閃光、飛蚊突然增多或視野異常，請及時尋求專業醫療評估。

## 已實現功能

- 在一個響應式頁面中呈現清晰的六階段流程：**進入 → 知情同意 → 校準 → 篩檢 → 解釋 → 指導**，並顯示進度和已用時間。
- 在申請攝影機權限前，通過必選且可撤銷的知情同意步驟說明用途和處理方式；是否儲存派生結果由獨立的可選同意控制。
- 支援攝影機權限、啟動、暫停/恢復、停止、退出及錯誤恢復提示；攝影機不可用時仍可通過手動模式完成視力測試。
- 使用 MediaPipe Face Landmarker 在本地推理；模型和 WASM 檔案隨專案提供，原始攝影機畫面不會被儲存或上傳。
- 基於 478 個臉部關鍵點進行眼周測量、即時光照檢查、已知 60 公分距離校準，並在 **40–80 公分**範圍內提供觀看距離指導。
- 當人臉丟失、頭部姿態不合格或即時攝影機條件不適合時自動暫停答題。
- 提供可選的連續監測模式，包含明確的啟動、暫停、恢復、停止和退出控制，以及可見狀態、有效時長、20 分鐘或停止/中斷摘要。普通瀏覽器在頁面隱藏時中斷；受信任的 Electron preload 橋接可明確允許窗口最小化或隱藏時繼續監測。
- 支援自適應眼睛縱橫比閾值、眨眼計數、每分鐘眨眼次數，以及結合眨眼率、距離行為和連續用屏時間的啟發式疲勞分數。
- 提供僅在當前會話內存在的症狀問卷，覆蓋視覺不適和警示症狀；症狀答案不會寫入瀏覽器儲存。
- 支援使用 ISO/IEC 7810 ID-1 實物（85.60 × 53.98 公釐，僅限確實符合該標準的銀行卡、身分證件或學生卡）或 ICAO TD-3 機讀護照（125 × 88 公釐）進行螢幕物理尺寸校準。沒有參照物時可使用明確標記為較低置信度的螢幕對角線估算。
- 交互式 Landolt C 篩檢包含從 20/200 到 20/20 的 9 個等級，每級 3 次測試，至少答對 2 次才能進入下一等級。
- 支援雙眼、左眼和右眼模式；可使用滑鼠、方向鍵或 WASD 作答。
- 提供三級解釋結果：**正常 / 注意 / 關注**；多個指標一致時會提高風險等級。
- 每次完成篩檢后嚴格按照四個編號部分解釋：結果、主要指標、置信度與局限、下一步。
- 在單屏結果卡中展示視力、眨眼率和觀看距離，並同時提供非診斷聲明和匹配的專業就醫建議。
- 提供可選的本地歷史、同眼趨勢、JSON 導出、可列印/分享摘要和本地 20 分鐘復測/休息提醒。每個結果使用不可變的完成快照；持久化摘要不包含症狀和逐題答案；演示結果不計入趨勢。
- 提供可配置的校園存取卡，可在本地生成QR 碼、復制連結、列印並連接專業轉介路徑，無需預約系統、特殊裝置或專人值守。
- 提供無攝影機演示模式以驗證產品流程；模擬結果會明確標記，且不會作為醫療證據展示。
- 支援選擇**簡體中文、繁體中文和英文**。首次使用時，如裝置/瀏覽器的首選語言屬於這些語言族，應用程式會自動匹配（`zh-Hans` 和大多數通用中文區域映射為簡體中文；`zh-Hant`、臺灣、香港和澳門映射為繁體中文）；無法識別或不支援時使用英文。使用者主動選擇的語言會儲存在本地，並在後續存取時優先使用。
- 提供響應式桌面和行動介面，包括適配安全區域的手機布局、豎屏攝影機區域、適合觸控的控制項和緊湊的橫屏測試布局；無需帳戶或應用伺服器。

## 技術棧

- React 18
- TypeScript 5
- Vite 5
- Electron 43（可選的 Windows/macOS 托盤外殼）
- MediaPipe Tasks Vision 0.10.35
- QRCode 1.5.4

應用程式採用純前端架構。攝影機畫面直接送入裝置上的 MediaPipe 推理，不會上傳到專案伺服器。若本地模型或 WASM 檔案缺失，程式碼保留靜態資源 CDN 回退；該回退只下載模型/執行時，不會上傳畫面。發布包已經包含所需模型和 WASM 檔案，正常本地執行不依賴此回退。

## 本地開發

### 環境要求

- Node.js 18.18 或更高版本
- npm
- 帶攝影機的電腦；沒有攝影機時可使用演示/手動模式
- 推薦使用當前版本的 Chrome/Edge 桌面瀏覽器、Android Chrome 或 iOS Safari

### 啟動

```bash
cd visionguard-ai
npm install
npm run dev
```

打開終端顯示的地址，通常為：

```text
http://127.0.0.1:5173
```

請使用 `http://127.0.0.1` 或 `http://localhost`，不要直接打開 `index.html`，因為瀏覽器安全策略可能阻止攝影機存取或 ES 模塊載入。

執行 `npm install` 后，資源準備腳本會：

- 將 MediaPipe WASM 檔案復制到 `public/mediapipe/wasm/`；
- 檢查 `public/models/face_landmarker.task`，僅在缺失時嘗試下載官方模型。

本交付版本已包含這兩類資源。

## 生產構建與執行

使用 Vite 構建和預覽：

```bash
npm run build
npm run preview
```

預設預覽地址為 `http://127.0.0.1:4173`。

交付內容也可包含預構建的 `dist/` 目錄。無需安裝 Node.js 即可執行：

```bash
python serve_dist.py
```

本地伺服器在 Windows 和 macOS 上優先使用 **Google Chrome**。它會檢查 `CHROME_PATH`、Chrome 的標準安裝目錄以及 `PATH` 中的命令；如果 Chrome 不可用或無法啟動，會安全回退到操作系統預設瀏覽器。無論打開哪個瀏覽器，伺服器都會繼續執行。

命令相對于終端的**當前目錄**解析，而不是相對于編輯器中打開的檔案。從專案目錄執行上述命令；從父級工作區執行時，可使用隨附的轉發啟動器或明確指定專案腳本：

```powershell
python .\serve_dist.py
# 從父目錄調用專案腳本：
python .\visionguard-ai\serve_dist.py
```

如果 Python 提示無法打開父目錄下的 `serve_dist.py`，請先執行 `cd .\visionguard-ai` 或使用明確路徑。IDE 的 **Ctrl+F5** 通常能夠執行，是因為編輯器會使用活動檔案的完整路徑。

選擇其他端口：

```bash
python serve_dist.py --port 8088
```

瀏覽器啟動選項：

```bash
# 明確保留 Chrome 優先策略（預設）
python serve_dist.py --browser chrome

# 跳過 Chrome 檢測，使用系統預設瀏覽器
python serve_dist.py --browser default

# 只啟動伺服器，不打開瀏覽器
python serve_dist.py --no-browser

# 檢視全部主機、端口和瀏覽器選項
python serve_dist.py --help
```

`--browser` 和 `--no-browser` 不能同時使用。如果 Chrome 位于非標準目錄，請先將 `CHROME_PATH` 設置為其可執行檔案完整路徑。

也可以使用任意靜態 HTTP 伺服器托管 `dist/`。請勿通過 `file://` 打開 `dist/index.html`。

### 可選的 Windows/macOS 桌面外殼

專案包含 Electron 托盤外殼，可在使用者同意后于窗口最小化或隱藏時持續監測。它只載入打包的本地 `dist/`，保持渲染進程不被節流，並且僅在監測活動期間阻止應用掛起。關閉窗口會將其隱藏；必須從托盤或 macOS 應用程式選單明確退出才能結束進程。

```bash
npm install
npm run desktop
```

有關安全邊界、攝影機權限、打包命令、簽名要求和后臺執行限制，請參閱 [DESKTOP.md](DESKTOP.md)。

### 行動裝置使用

攝影機請求優先選擇前置攝影機，界面分別適配窄豎屏和低高度橫屏。手機必須通過 HTTPS 部署存取應用（開發期間也可使用手機自身的 `localhost`）；其他電腦的 `127.0.0.1` 無法從手機存取。使用實體卡/護照校準時，如果窄屏無法完整容納輪廓，可將手機旋轉為橫屏，或使用明確標記的螢幕對角線估算。仍需在每種支援的手機/瀏覽器組合上驗收真實攝影機性能和物理精度。

### 校園部署配置

可以通過已發布 URL 的查詢參數加入機構資訊，使協調人員無需重新構建應用即可生成校園QR 碼/連結：

| 查詢參數 | 用途 | 限制 |
|---|---|---|
| `campus` | 校園或支援部門名稱 | 80 個字符 |
| `campaign` | 活動標題 | 120 個字符 |
| `code` | 印在存取卡上的可選程式碼 | 32 個字符 |
| `referral` | 經批準的 `https:`、`http:` 或 `mailto:` 轉介地址 | 2,048 個字符 |

示例：

```text
https://screening.example.edu/?campus=North%20Campus&campaign=Healthy%20Eyes%20Week&code=EYE-2026&referral=https%3A%2F%2Fhealth.example.edu%2Feye-care
```

無效、超長、相對、包含憑據或不安全的轉介參數會被忽略，並繼續使用本地化的應用預設值。URL 參數只配置公開活動資訊，不包含或傳輸篩檢結果。

有關推薦的 HTTPS 靜態托管、臨時隧道、校園內網方案、PWA/原生升級路徑，以及為什么其他手機無法打開 localhost QR 碼，請參閱 [DEPLOYMENT_QR.md](DEPLOYMENT_QR.md)。

## 使用方法

1. 直接打開頁面或掃描校園QR 碼/連結。首次使用時，界面會匹配支援的裝置/瀏覽器語言，否則使用英文；可通過始終可見的頁首選擇器在簡體中文、繁體中文和英文之間切換。
2. 閱讀隱私說明並同意使用攝影機。是否儲存結果歷史是獨立的可選項，且可以撤銷。
3. 啟動攝影機檢測並允許存取，或選擇無攝影機演示/手動路徑。
4. 保持臉部位于畫面中，根據提示改善光照，將眼睛置于約 60 公分處並執行距離校準。監測期間保持在 40–80 公分範圍內。
5. 選擇螢幕尺寸校準方式。將螢幕輪廓與符合 ISO/IEC 7810 ID-1 的銀行卡/身分證件/學生卡或 ICAO TD-3 護照匹配並儲存。沒有參照物時，輸入當前顯示裝置的對角線尺寸，並將結果儲存為近似校準。
6. 選擇適用症狀，再選擇雙眼、左眼或右眼篩檢。單眼測試時輕輕遮住另一只眼睛，不要按壓眼球。
7. 判斷 Landolt C 的缺口方向。可隨時暫停、停止或退出。監測無法繼續時可使用手動回退完成視力答題。
8. 檢視風險等級和四部分解釋。可列印/復制分享摘要、檢視同眼趨勢、設置本地 20 分鐘提醒、導出已同意儲存的歷史，或使用專業/校園轉介指導。

演示結果會明確標記為模擬資料。

### 可選連續監測

同意使用攝影機后，在專用面板中選擇**啟動連續監測**。普通瀏覽器中，該功能只能在前臺執行，並非操作系統后臺服務：僅在頁面打開且可見、即時攝影機流活動時計時。暫停會停止攝影機采集；停止或退出會結束當前監測窗口並生成結構化摘要。完成 20 分鐘窗口時也會自動生成同類摘要。

Electron 桌面外殼可以暴露最小化 preload 契約 `window.desktopRuntime = { isDesktop: true, platform, setMonitoringActive(boolean) }`。只有檢測到該受信任橋接時，窗口最小化或隱藏後才能繼續監測；點擊桌面窗口關閉按鈕只會隱藏到托盤，不會中斷監測。渲染進程仍需要活動媒體流，並且僅在連續監測確實活動時通知橋接。明確退出應用、重新載入或導航渲染頁面、凍結、長時間執行間隔或媒體流丟失都會中斷窗口。普通瀏覽器在頁面隱藏時始終中斷，絕不會根據 User-Agent 推斷桌面能力。報告不包含原始畫面；除非使用者同意本地儲存，否則報告只保留在頁面內存中。

## 核心計算

### 相對距離

在 60 公分參考位置，外眼角間距除以攝影機畫面的固有寬度，得到與分辨率無關的比例 `R_ref`。後續比例為 `R_now` 時：

```text
D_now = D_ref × R_ref / R_now
```

僅在臉部姿態和光照合格時發布估算值，並使用最近有效樣本的滾動中位數進行平滑。攝影機畫面方向或寬高比變化會使基線失效並要求重新校準。該估算仍假定焦距、數位變焦、裁剪和臉部條件保持穩定，不能代替深度攝影機或經過校準的測距裝置。

### 眨眼

應用使用每只眼睛的六個關鍵點計算眼睛縱橫比（EAR），建立個人睜眼基線，並以該基線的一定比例作為閉眼閾值。只有閉眼后重新睜開且持續時間處于合理範圍時才計為一次眨眼。

### 疲勞分數

疲勞分數是 0–100 的啟發式行為指標，輸入包括：

- 低于目標的眨眼率；
- 監測時間中偏離預期距離行為的比例；
- 連續監測時長。

該分數沒有臨床診斷意義。監測初期會顯示“正在收集資料”，避免根據極短樣本得出結論。

### 視標尺寸

實體參照物校準根據 ISO/IEC 7810 ID-1 輪廓（85.60 × 53.98 公釐）或 ICAO TD-3 機讀護照輪廓（125 × 88 公釐）確定每公釐對應的 CSS 像素。不同國家和簽發機構的身分證件與學生卡規格並不一致，因此只有明確符合 ID-1 時才能使用卡片選項。Landolt C 高度采用目標 Snellen 等級對應的 5 弧分視角，並換算為螢幕像素。

無參照物方式要求輸入當前裝置的物理對角線尺寸，並結合 `window.screen.width` 和 `window.screen.height` 估算每公釐邏輯螢幕像素。手機上不會預填筆記本尺寸。橫豎屏交換寬高軸不會改變對角線計算，但系統/界面縮放、多顯示器切換和錯誤的裝置規格會影響結果。

瀏覽器無法可靠自動檢測顯示裝置的物理尺寸。標準 Web API 提供 CSS 像素尺寸，卻無法提供可信的面板物理寬高或顯示器 EDID。同樣，普通 `getUserMedia()` 攝影機元資料不包含從單目圖像恢復絕對尺寸所需的傳感器尺寸、焦距和場景尺度校準資訊。因此，僅使用攝影機分辨率或 `devicePixelRatio` 不能替代校準。獲得顯示硬件元資料權限的原生應用、深度/AR API 或畫面中的已知物體可以提供更強的裝置特定校準，但本瀏覽器 MVP 不聲稱具備該精度。

更換顯示器、瀏覽器縮放、螢幕縮放或顯示方向/設置后，請重新校準。

## 原生應用安裝包

- Windows：`npm run desktop:dist:win` 創建 NSIS 安裝程序。
- Android：安裝 JDK 21 和 Android SDK 36 后，`npm run android:apk` 創建 debug 簽名、可直接安裝的 APK。
- Linux：`npm run desktop:dist:linux` 創建 AppImage 和 deb 包。
- macOS：在 macOS 上執行 `npm run desktop:dist:mac`，分別創建 Intel 和 Apple Silicon DMG。

Android 權限、簽名和裝置測試見 [ANDROID.md](ANDROID.md)；桌面執行時、簽名和打包細節見 [DESKTOP.md](DESKTOP.md)。倉庫級 `Package applications` GitHub Actions 工作流可在手動觸發或推送 `v*` 標簽時，在各原生宿主上構建未簽名/測試產物。

## 隱私與資料

- 攝影機畫面不會被儲存，也不會發送到應用伺服器。
- 僅在使用者明確同意用途和處理方式后申請攝影機權限，並可在界面中暫停或停止。
- 臉部關鍵點和派生指標在本地處理；原始圖片和影片從不寫入本地歷史。
- 症狀答案僅在當前會話存在，不會持久化。
- 只有使用者選擇可選的儲存同意后，派生篩檢結果和提醒設置才會儲存在當前瀏覽器的 `localStorage`。顯示校準和語言屬於本地設置，不是原始健康媒體。
- 連續監測報告只包含時間戳、活動/追蹤時長、追蹤覆蓋率、眨眼率、距離、過近比例、疲勞指標和語義建議程式碼，並僅在同一可選儲存同意下持久化。
- 界面提供刪除本地資料的控制；清除瀏覽器站點資料也會刪除本地記錄。
- 僅在使用者主動請求時執行 JSON 導出，並只儲存到使用者選擇的位置。
- 程式碼不包含登錄、遠程資料庫、分析追蹤或廣告 SDK。

## 已知局限

- 單目攝影機只能提供經過校準的相對距離估算；姿態、鏡頭視場、自動對焦和數位變焦都會影響結果。
- 眼鏡反光、遮擋、低光、強背光、攝影機分辨率和快速頭部移動會降低關鍵點穩定性。
- 螢幕物理尺寸校準依賴正確選擇和放置參照物；並非所有身分證件或學生卡都符合 ID-1。僅對角線方式是近似估算，在瀏覽器/系統縮放或多顯示器設置下可能不準確；相關顯示或縮放變化后兩種方式都需要重新校準。
- 家用螢幕視力篩檢會受環境、遮眼方法、學習效應和顯示品質影響，只能提示可能需要進一步檢查。
- 疲勞分數和風險等級是透明的規則型篩檢輔助，不是診斷模型或經過驗證的醫療裝置。
- 瀏覽器頁面關閉後，或桌面渲染進程/應用程式明確退出後，連續監測無法繼續。Electron 外殼中，窗口關閉按鈕只隱藏到托盤，監測可以繼續；渲染頁面重新載入/導航或明確退出應用程式會中斷。普通瀏覽器在頁面隱藏時中斷。瀏覽器或渲染進程凍結、定時器節流、裝置休眠、攝影機斷開和操作系統電源策略仍可能中斷監測；這不是系統級監測。
- 專案的數值精度目標——視力誤差在一行以內、眨眼檢測 F1 高于 0.85、40–80 公分範圍內距離誤差不超過 5 公分——**尚未**通過真實參與者和校準參照測試建立。通過單元測試和瀏覽器測試並不能證明這些指標。
- 校園 URL 和轉介內容是可配置占位內容；真實部署必須填入機構批準的頁面、診所聯系方式、升級處置文案和隱私政策。
