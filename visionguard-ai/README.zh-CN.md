# VisionGuard AI

[English](README.md) | **简体中文** | [繁體中文](README.zh-TW.md) | [仓库首页](../README.md)

VisionGuard AI 是一款基于浏览器的视力筛查与数字视疲劳监测 MVP。它使用设备摄像头在本地执行面部和眼部关键点推理，估算相对观看距离、统计眨眼、提供启发式疲劳指标，并引导用户完成 Landolt C 缺口方向测试。

> **重要提示：** 本项目仅用于健康筛查和产品演示，不能诊断近视、散光、眼病或其他医疗状况，也不能代替验光师或眼科医生的检查。如持续出现视物模糊、眼痛、复视、闪光、飞蚊突然增多或视野异常，请及时寻求专业医疗评估。

## 已实现功能

- 在一个响应式页面中呈现清晰的六阶段流程：**进入 → 知情同意 → 校准 → 筛查 → 解释 → 指导**，并显示进度和已用时间。
- 在申请摄像头权限前，通过必选且可撤销的知情同意步骤说明用途和处理方式；是否保存派生结果由独立的可选同意控制。
- 支持摄像头权限、启动、暂停/恢复、停止、退出及错误恢复提示；摄像头不可用时仍可通过手动模式完成视力测试。
- 使用 MediaPipe Face Landmarker 在本地推理；模型和 WASM 文件随项目提供，原始摄像头画面不会被保存或上传。
- 基于 478 个面部关键点进行眼周测量、实时光照检查、已知 60 厘米距离校准，并在 **40–80 厘米**范围内提供观看距离指导。
- 当人脸丢失、头部姿态不合格或实时摄像头条件不适合时自动暂停答题。
- 提供可选的连续监测模式，包含明确的启动、暂停、恢复、停止和退出控制，以及可见状态、有效时长、20 分钟或停止/中断摘要。普通浏览器在页面隐藏时中断；受信任的 Electron preload 桥接可明确允许窗口最小化或隐藏时继续监测。
- 支持自适应眼睛纵横比阈值、眨眼计数、每分钟眨眼次数，以及结合眨眼率、距离行为和连续用屏时间的启发式疲劳分数。
- 提供仅在当前会话内存在的症状问卷，覆盖视觉不适和警示症状；症状答案不会写入浏览器存储。
- 支持使用 ISO/IEC 7810 ID-1 实物（85.60 × 53.98 毫米，仅限确实符合该标准的银行卡、身份证件或学生卡）或 ICAO TD-3 机读护照（125 × 88 毫米）进行屏幕物理尺寸校准。没有参照物时可使用明确标记为较低置信度的屏幕对角线估算。
- 交互式 Landolt C 筛查包含从 20/200 到 20/20 的 9 个等级，每级 3 次测试，至少答对 2 次才能进入下一等级。
- 支持双眼、左眼和右眼模式；可使用鼠标、方向键或 WASD 作答。
- 提供三级解释结果：**正常 / 注意 / 关注**；多个指标一致时会提高风险等级。
- 每次完成筛查后严格按照四个编号部分解释：结果、主要指标、置信度与局限、下一步。
- 在单屏结果卡中展示视力、眨眼率和观看距离，并同时提供非诊断声明和匹配的专业就医建议。
- 提供可选的本地历史、同眼趋势、JSON 导出、可打印/分享摘要和本地 20 分钟复测/休息提醒。每个结果使用不可变的完成快照；持久化摘要不包含症状和逐题答案；演示结果不计入趋势。
- 提供可配置的校园访问卡，可在本地生成二维码、复制链接、打印并连接专业转介路径，无需预约系统、特殊设备或专人值守。
- 提供无摄像头演示模式以验证产品流程；模拟结果会明确标记，且不会作为医疗证据展示。
- 支持选择**简体中文、繁体中文和英文**。首次使用时，如设备/浏览器的首选语言属于这些语言族，应用会自动匹配（`zh-Hans` 和大多数通用中文区域映射为简体中文；`zh-Hant`、台湾、香港和澳门映射为繁体中文）；无法识别或不支持时使用英文。用户主动选择的语言会保存在本地，并在后续访问时优先使用。
- 提供响应式桌面和移动界面，包括适配安全区域的手机布局、竖屏摄像头区域、适合触控的控件和紧凑的横屏测试布局；无需账户或应用服务器。

## 技术栈

- React 18
- TypeScript 5
- Vite 5
- Electron 43（可选的 Windows/macOS 托盘外壳）
- MediaPipe Tasks Vision 0.10.35
- QRCode 1.5.4
- Vitest

应用采用纯前端架构。摄像头画面直接送入设备上的 MediaPipe 推理，不会上传到项目服务器。若本地模型或 WASM 文件缺失，代码保留静态资源 CDN 回退；该回退只下载模型/运行时，不会上传画面。发布包已经包含所需模型和 WASM 文件，正常本地运行不依赖此回退。

## 本地开发

### 环境要求

- Node.js 18.18 或更高版本
- npm
- 带摄像头的电脑；没有摄像头时可使用演示/手动模式
- 推荐使用当前版本的 Chrome/Edge 桌面浏览器、Android Chrome 或 iOS Safari

### 启动

```bash
cd visionguard-ai
npm install
npm run dev
```

打开终端显示的地址，通常为：

```text
http://127.0.0.1:5173
```

请使用 `http://127.0.0.1` 或 `http://localhost`，不要直接打开 `index.html`，因为浏览器安全策略可能阻止摄像头访问或 ES 模块加载。

执行 `npm install` 后，资源准备脚本会：

- 将 MediaPipe WASM 文件复制到 `public/mediapipe/wasm/`；
- 检查 `public/models/face_landmarker.task`，仅在缺失时尝试下载官方模型。

本交付版本已包含这两类资源。

## 生产构建与运行

使用 Vite 构建和预览：

```bash
npm run build
npm run preview
```

默认预览地址为 `http://127.0.0.1:4173`。

交付内容也可包含预构建的 `dist/` 目录。无需安装 Node.js 即可运行：

```bash
python serve_dist.py
```

本地服务器在 Windows 和 macOS 上优先使用 **Google Chrome**。它会检查 `CHROME_PATH`、Chrome 的标准安装目录以及 `PATH` 中的命令；如果 Chrome 不可用或无法启动，会安全回退到操作系统默认浏览器。无论打开哪个浏览器，服务器都会继续运行。

命令相对于终端的**当前目录**解析，而不是相对于编辑器中打开的文件。从项目目录运行上述命令；从父级工作区运行时，可使用随附的转发启动器或明确指定项目脚本：

```powershell
python .\serve_dist.py
# 从父目录调用项目脚本：
python .\visionguard-ai\serve_dist.py
```

如果 Python 提示无法打开父目录下的 `serve_dist.py`，请先执行 `cd .\visionguard-ai` 或使用明确路径。IDE 的 **Ctrl+F5** 通常能够运行，是因为编辑器会使用活动文件的完整路径。

选择其他端口：

```bash
python serve_dist.py --port 8088
```

浏览器启动选项：

```bash
# 明确保留 Chrome 优先策略（默认）
python serve_dist.py --browser chrome

# 跳过 Chrome 检测，使用系统默认浏览器
python serve_dist.py --browser default

# 只启动服务器，不打开浏览器
python serve_dist.py --no-browser

# 查看全部主机、端口和浏览器选项
python serve_dist.py --help
```

`--browser` 和 `--no-browser` 不能同时使用。如果 Chrome 位于非标准目录，请先将 `CHROME_PATH` 设置为其可执行文件完整路径。

也可以使用任意静态 HTTP 服务器托管 `dist/`。请勿通过 `file://` 打开 `dist/index.html`。

### 可选的 Windows/macOS 桌面外壳

项目包含 Electron 托盘外壳，可在用户同意后于窗口最小化或隐藏时持续监测。它只加载打包的本地 `dist/`，保持渲染进程不被节流，并且仅在监测活动期间阻止应用挂起。关闭窗口会将其隐藏；必须从托盘或 macOS 应用菜单明确退出才能结束进程。

```bash
npm install
npm run desktop
```

### 移动端使用

摄像头请求优先选择前置摄像头，界面分别适配窄竖屏和低高度横屏。手机必须通过 HTTPS 部署访问应用（开发期间也可使用手机自身的 `localhost`）；其他电脑的 `127.0.0.1` 无法从手机访问。使用实体卡/护照校准时，如果窄屏无法完整容纳轮廓，可将手机旋转为横屏，或使用明确标记的屏幕对角线估算。仍需在每种支持的手机/浏览器组合上验收真实摄像头性能和物理精度。

### 校园部署配置

可以通过已发布 URL 的查询参数加入机构信息，使协调人员无需重新构建应用即可生成校园二维码/链接：

| 查询参数 | 用途 | 限制 |
|---|---|---|
| `campus` | 校园或支持部门名称 | 80 个字符 |
| `campaign` | 活动标题 | 120 个字符 |
| `code` | 印在访问卡上的可选代码 | 32 个字符 |
| `referral` | 经批准的 `https:`、`http:` 或 `mailto:` 转介地址 | 2,048 个字符 |

示例：

```text
https://screening.example.edu/?campus=North%20Campus&campaign=Healthy%20Eyes%20Week&code=EYE-2026&referral=https%3A%2F%2Fhealth.example.edu%2Feye-care
```

无效、超长、相对、包含凭据或不安全的转介参数会被忽略，并继续使用本地化的应用默认值。URL 参数只配置公开活动信息，不包含或传输筛查结果。

## 使用方法

1. 直接打开页面或扫描校园二维码/链接。首次使用时，界面会匹配支持的设备/浏览器语言，否则使用英文；可通过始终可见的页首选择器在简体中文、繁体中文和英文之间切换。
2. 阅读隐私说明并同意使用摄像头。是否保存结果历史是独立的可选项，且可以撤销。
3. 启动摄像头检测并允许访问，或选择无摄像头演示/手动路径。
4. 保持面部位于画面中，根据提示改善光照，将眼睛置于约 60 厘米处并执行距离校准。监测期间保持在 40–80 厘米范围内。
5. 选择屏幕尺寸校准方式。将屏幕轮廓与符合 ISO/IEC 7810 ID-1 的银行卡/身份证件/学生卡或 ICAO TD-3 护照匹配并保存。没有参照物时，输入当前显示设备的对角线尺寸，并将结果保存为近似校准。
6. 选择适用症状，再选择双眼、左眼或右眼筛查。单眼测试时轻轻遮住另一只眼睛，不要按压眼球。
7. 判断 Landolt C 的缺口方向。可随时暂停、停止或退出。监测无法继续时可使用手动回退完成视力答题。
8. 查看风险等级和四部分解释。可打印/复制分享摘要、查看同眼趋势、设置本地 20 分钟提醒、导出已同意保存的历史，或使用专业/校园转介指导。

演示结果会明确标记为模拟数据。

### 可选连续监测

同意使用摄像头后，在专用面板中选择**启动连续监测**。普通浏览器中，该功能只能在前台运行，并非操作系统后台服务：仅在页面打开且可见、实时摄像头流活动时计时。暂停会停止摄像头采集；停止或退出会结束当前监测窗口并生成结构化摘要。完成 20 分钟窗口时也会自动生成同类摘要。

Electron 桌面外壳可以暴露最小化 preload 契约 `window.desktopRuntime = { isDesktop: true, platform, setMonitoringActive(boolean) }`。只有检测到该受信任桥接时，窗口最小化或隐藏后才能继续监测；点击桌面窗口关闭按钮只会隐藏到托盘，不会中断监测。渲染进程仍需要活动媒体流，并且仅在连续监测确实活动时通知桥接。明确退出应用、重新加载或导航渲染页面、冻结、长时间执行间隔或媒体流丢失都会中断窗口。普通浏览器在页面隐藏时始终中断，绝不会根据 User-Agent 推断桌面能力。报告不包含原始画面；除非用户同意本地存储，否则报告只保留在页面内存中。

## 核心计算

### 相对距离

在 60 厘米参考位置，外眼角间距除以摄像头画面的固有宽度，得到与分辨率无关的比例 `R_ref`。后续比例为 `R_now` 时：

```text
D_now = D_ref × R_ref / R_now
```

仅在面部姿态和光照合格时发布估算值，并使用最近有效样本的滚动中位数进行平滑。摄像头画面方向或宽高比变化会使基线失效并要求重新校准。该估算仍假定焦距、数字变焦、裁剪和面部条件保持稳定，不能代替深度摄像头或经过校准的测距设备。

### 眨眼

应用使用每只眼睛的六个关键点计算眼睛纵横比（EAR），建立个人睁眼基线，并以该基线的一定比例作为闭眼阈值。只有闭眼后重新睁开且持续时间处于合理范围时才计为一次眨眼。

### 疲劳分数

疲劳分数是 0–100 的启发式行为指标，输入包括：

- 低于目标的眨眼率；
- 监测时间中偏离预期距离行为的比例；
- 连续监测时长。

该分数没有临床诊断意义。监测初期会显示“正在收集数据”，避免根据极短样本得出结论。

### 视标尺寸

实体参照物校准根据 ISO/IEC 7810 ID-1 轮廓（85.60 × 53.98 毫米）或 ICAO TD-3 机读护照轮廓（125 × 88 毫米）确定每毫米对应的 CSS 像素。不同国家和签发机构的身份证件与学生卡规格并不一致，因此只有明确符合 ID-1 时才能使用卡片选项。Landolt C 高度采用目标 Snellen 等级对应的 5 角分视角，并换算为屏幕像素。

无参照物方式要求输入当前设备的物理对角线尺寸，并结合 `window.screen.width` 和 `window.screen.height` 估算每毫米逻辑屏幕像素。手机上不会预填笔记本尺寸。横竖屏交换宽高轴不会改变对角线计算，但系统/界面缩放、多显示器切换和错误的设备规格会影响结果。

浏览器无法可靠自动检测显示设备的物理尺寸。标准 Web API 提供 CSS 像素尺寸，却无法提供可信的面板物理宽高或显示器 EDID。同样，普通 `getUserMedia()` 摄像头元数据不包含从单目图像恢复绝对尺寸所需的传感器尺寸、焦距和场景尺度校准信息。因此，仅使用摄像头分辨率或 `devicePixelRatio` 不能替代校准。获得显示硬件元数据权限的原生应用、深度/AR API 或画面中的已知物体可以提供更强的设备特定校准，但本浏览器 MVP 不声称具备该精度。

更换显示器、浏览器缩放、屏幕缩放或显示方向/设置后，请重新校准。

## 原生应用安装包

- Windows：`npm run desktop:dist:win` 创建 NSIS 安装程序。
- Android：安装 JDK 21 和 Android SDK 36 后，`npm run android:apk` 创建 debug 签名、可直接安装的 APK。
- Linux：`npm run desktop:dist:linux` 创建 AppImage 和 deb 包。
- macOS：在 macOS 上运行 `npm run desktop:dist:mac`，分别创建 Intel 和 Apple Silicon DMG。

仓库级 `Package applications` GitHub Actions 工作流可在手动触发或推送 `v*` 标签时，在各原生宿主上构建未签名/测试产物。

## 隐私与数据

- 摄像头画面不会被存储，也不会发送到应用服务器。
- 仅在用户明确同意用途和处理方式后申请摄像头权限，并可在界面中暂停或停止。
- 面部关键点和派生指标在本地处理；原始图片和视频从不写入本地历史。
- 症状答案仅在当前会话存在，不会持久化。
- 只有用户选择可选的存储同意后，派生筛查结果和提醒设置才会保存在当前浏览器的 `localStorage`。显示校准和语言属于本地设置，不是原始健康媒体。
- 连续监测报告只包含时间戳、活动/追踪时长、追踪覆盖率、眨眼率、距离、过近比例、疲劳指标和语义建议代码，并仅在同一可选存储同意下持久化。
- 界面提供删除本地数据的控制；清除浏览器站点数据也会删除本地记录。
- 仅在用户主动请求时执行 JSON 导出，并只保存到用户选择的位置。
- 代码不包含登录、远程数据库、分析追踪或广告 SDK。

## 已知局限

- 单目摄像头只能提供经过校准的相对距离估算；姿态、镜头视场、自动对焦和数字变焦都会影响结果。
- 眼镜反光、遮挡、低光、强背光、摄像头分辨率和快速头部移动会降低关键点稳定性。
- 屏幕物理尺寸校准依赖正确选择和放置参照物；并非所有身份证件或学生卡都符合 ID-1。仅对角线方式是近似估算，在浏览器/系统缩放或多显示器设置下可能不准确；相关显示或缩放变化后两种方式都需要重新校准。
- 家用屏幕视力筛查会受环境、遮眼方法、学习效应和显示质量影响，只能提示可能需要进一步检查。
- 疲劳分数和风险等级是透明的规则型筛查辅助，不是诊断模型或经过验证的医疗设备。
- 浏览器页面关闭后，或桌面渲染进程/应用明确退出后，连续监测无法继续。Electron 外壳中，窗口关闭按钮只隐藏到托盘，监测可以继续；渲染页面重新加载/导航或明确退出应用会中断。普通浏览器在页面隐藏时中断。浏览器或渲染进程冻结、定时器节流、设备休眠、摄像头断开和操作系统电源策略仍可能中断监测；这不是系统级监测。
- 项目的数值精度目标——视力误差在一行以内、眨眼检测 F1 高于 0.85、40–80 厘米范围内距离误差不超过 5 厘米——**尚未**通过真实参与者和校准参照测试建立。通过单元测试和浏览器测试并不能证明这些指标。
- 校园 URL 和转介内容是可配置占位内容；真实部署必须填入机构批准的页面、诊所联系方式、升级处置文案和隐私政策。
